# Repo Role: AgentTools

Status: active toolbelt repo.

This repository is not a Dream.OS variant. Its job is to hold reusable agent/operator tooling that can be called from DreamVault, Cursor lanes, Termux lanes, or project-specific repos.

## Role

- Reusable automation utilities
- Agent execution helpers
- CLI/helper scripts that are not DreamVault runtime SSOT
- Salvage destination for generic tools that should not live inside product repos

## Boundary

AgentTools should not become the canonical Dream.OS runtime. DreamVault remains the operational control and task SSOT.

## Promotion rule

A tool may be promoted here only when:

1. It is reusable outside one project.
2. It has no hardcoded secrets.
3. It has a smoke test, usage note, or clear command example.
4. It does not duplicate an existing DreamVault runtime tool without a deprecation note.
